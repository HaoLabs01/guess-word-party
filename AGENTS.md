# Agent 开发准则

本项目的最高目标来自 `docs/app-flow-spec.md` 和 `docs/gameplay-spec.md`。任何 agent 在修改 App、文档或测试前，都要先用这些目标判断优先级。

## 最高目的

- 指哪儿打哪儿：用户做出的点头、抬头、点击或开关操作，必须产生符合直觉的结果。
- 误触率要降到最低：宁可要求动作更明确，也不要让正常手持晃动、交接手机或回正过程触发得分/跳过。
- 最快看懂玩法：第一次玩的描述人和猜词人，应能在准备页和 5 秒倒计时内理解核心规则。
- 不会错误操作：开始后不再要求猜词人看屏幕、点屏幕或处理权限；所有需要确认的操作都必须前置。

当界面美观、功能数量、录制便利性与这些目标冲突时，优先保护这些目标。

## 修改前检查

- 先阅读 `docs/gameplay-spec.md`，确认线下玩法、手机姿态和动作含义。
- 再阅读 `docs/app-flow-spec.md`，确认当前页面所处流程的验收标准。
- 对会影响规则、流程或提示的改动，补充自动测试。
- 开始游戏后不得新增授权弹窗、复杂设置或需要猜词人阅读的说明。
