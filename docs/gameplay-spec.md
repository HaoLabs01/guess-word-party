# 游戏玩法验收标准

这份文档定义线下猜词游戏本身的规则。后续界面文案、动作识别和反馈都必须与这里保持一致。

## 角色

- 猜词人：手持手机，屏幕朝向自己但不主动看题词，通过听描述来猜。
- 描述人：站在屏幕对面，看屏幕上的题词并描述。
- 主持/操作者：在开始前选择词库、时间、动作权限和录制开关；开始后不应再要求猜词人看屏幕或点屏幕。

## 手机姿态

- 标准姿态：屏幕朝向猜词人，手机贴近额头或举在额前。
- 点头=屏幕向下扣=猜对：计分 +1，并切换下一个词。
- 抬头=屏幕向天翻=跳过：不计分，并切换下一个词。
- 动作判断以屏幕这一面为准，不以手机背面为准。
- 为避免误触，触发后必须回到接近中立位，才能识别下一次动作。

## 描述规则

- 描述人可以描述意思、特征、场景、动作、用途、联想或相近概念。
- 描述人不能说出题面中的字，也不能把题词拆字或直接读音提示。
- 题词只给描述人看；倒计时和准备阶段不能提前露出第一题。

## 计分规则

- 猜对 +1。
- 跳过不加分。
- 时间结束后立即停止本局。
- 手动停止本局时，保留当前得分并进入结束页。

## 验收标准

- 任何界面文案都必须清楚表达“点头=屏幕向下扣=猜对”和“抬头=屏幕向天翻=跳过”。
- 游戏开始后不得再出现需要猜词人阅读或点击的授权流程。
- 倒计时期间不得显示题词。
- 动作反馈必须短、明确，能让手持设备的人知道当前动作是否被识别。
- 键盘快捷键只能作为调试或桌面辅助，不得改变线下手机动作玩法。
