# App 优化 Review

本 Review 以 `gameplay-spec.md` 和 `app-flow-spec.md` 为验收标准，对当前 App 做差距检查。

## 必须改

- 建立玩法与 App 流程两份验收标准文档，避免后续优化只凭感觉改动。
- README 改为项目入口，并链接到验收标准文档。
- 倒计时提示压缩为三短句：本局词库、别说题面字、点头/抬头含义。

## 建议改

- 准备页“开启动作”改为“开启动作权限”，让用户知道这是授权步骤。
- 保留当前 5 秒倒计时和不露题逻辑，避免猜词人在交接手机时看到第一题。
- 保留当前动作反馈短词，降低手持设备的人理解成本。

## 暂不改

- 暂不继续收紧动作阈值，避免把正常点头/抬头变得太难触发。
- 暂不新增教程页、音效或复杂引导，以免增加开始前负担。
- 暂不改变录像保存策略；线上持久化仍作为后续独立需求。

## 第一轮执行范围

- 执行“必须改”中的文档、README 和倒计时提示优化。
- 执行低风险“建议改”：准备页动作按钮文案。
- 不改计分规则、动作阈值、录像格式和保存逻辑。
